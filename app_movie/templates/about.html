1. Django Setup

Installation:

First, you install Django using pip (pip install django).

Project Creation:

Create a new Django project (django-admin startproject jk_movie_blog).

App Creation:

Inside your project, create a Django app for your movie-related functionality (python manage.py startapp movies).



2. Static Files

Purpose: Static files are used for CSS, JavaScript, images, etc., to style and enhance the frontend of your website.

Configuration:

Ensure 'django.contrib.staticfiles' is in your INSTALLED_APPS in settings.py. Define STATIC_URL and either STATICFILES_DIRS for development or STATIC_ROOT for production to specify where Django should look for static files.

Usage:

Place your CSS file (trailer.css, for example) in a directory named static inside your app (movies/static/css/trailer.css). Link to this CSS file in your templates using {% static 'css/trailer.css' %}.



3. Dynamic Functionality

Models:

Define Django models in movies/models.py for entities like Movie with fields such as title, trailer_link, description, etc.

Views:

Create views (movies/views.py) to handle requests related to movies. Implement functions to fetch data from the database and render templates dynamically.

Templates:

Create HTML templates (templates directory inside your app) for rendering pages. Extend base templates (navigation.html, for example) to maintain consistent structure across pages.

Routing:

Define URL patterns (movies/urls.py) to map URLs to views. Include these URLs in the project's main urls.py (urlpatterns).
